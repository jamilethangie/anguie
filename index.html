Diseño y Desarrollo Web - 1er Trimestre: ¿Qué es GitHub?

GitHub es una plataforma de desarrollo colaborativo de software que utiliza Git para el control de versiones. Es ampliamente utilizado por desarrolladores para alojar sus proyectos, colaborar en código con otros, y gestionar cambios de manera eficiente.

Conceptos clave de GitHub:

  • Repositorio (Repository): Es el lugar donde se almacena tu proyecto, incluyendo todos los archivos, el historial de versiones y la información de colaboración.
  • Git: Es el sistema de control de versiones distribuido que GitHub utiliza. Te permite rastrear cambios en tu código, volver a versiones anteriores y trabajar en paralelo con otros.
  • Commit: Un "commit" es una instantánea de los cambios en tu repositorio en un momento dado. Cada commit tiene un mensaje que describe los cambios realizados.
  • Branch (Rama): Las ramas te permiten trabajar en diferentes versiones de un proyecto simultáneamente. Puedes crear una rama para nuevas características o correcciones de errores sin afectar el código principal.
  • Merge (Fusión): Es el proceso de integrar los cambios de una rama a otra, generalmente de una rama de características a la rama principal.
  • Pull Request (Solicitud de Extracción): Es una forma de notificar a otros desarrolladores que has completado una característica o corrección y que te gustaría que revisaran tu código antes de fusionarlo con la rama principal. Facilita la revisión de código y la discusión.
  • Fork: Crear un "fork" de un repositorio es hacer una copia de este en tu propia cuenta de GitHub, lo que te permite experimentar con el proyecto sin afectar el original.

GitHub no solo es una herramienta para programadores, sino también una comunidad donde se puede descubrir, contribuir y aprender de proyectos de código abierto de todo el mundo.
